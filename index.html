核心目標

    • 用於長期整體規劃
  • Hot Scheduling（熱排程）
    • 依 X-band contact 建立 GAT / GOHOME 活動
    • 用於動態調整
  • 微調功能（5 種）
    • runUIReHot、addResourceChange
    • adjustActivity、runUIReFilterHot1 / Hot2

肆、數學模型與最佳化目標
MATHEMATICAL MODEL MILP / CPLEX
決策變數與目標函數
  • 決策變數
    • 二元變數：各活動是否執行
    • 整數變數：RSI PBK / DEL 起始時間
    • 站點選擇變數：地面站指派
  • 目標函數（最大化）
    • 可執行活動總數最大
    • 選擇 prefer site 加權
    • RSI PBK / DEL 越早完成越好
  • CPLEX 求解器調優
    • Threads：8
    • Gomory / Cliques Cuts：Aggressive
    • NodeSelect：Best-Estimate Search
    • BigM ≥ 1,000,000（5 天 × 86400）

前言目標
  • 建立衛星排程最佳化模型
  • 唯一使用 ILOG CPLEX 的部分

活動核心限制

  • RSI PBK 須在 11.5° X-band 時窗

  • 同 entry ID 活動需按序執行
  • 記憶體不足時 REC 不可執行

  • 可對衛星軌道與地面站規劃
  • 可解決排程衝突
  • 可同時支援多衛星操作
  • 可支援長短期規劃與調度
  • 可支援科學資料收集
  • 可進行快速重新規劃
  • 可支援發射與緊急規劃

柒、第三階段　測試問題修訂（15 項）
PHASE 03 — TESTING & REVISION 2024 / 05 / 08
1 系統產生的 GAT 影響求解結果 X-band contact 起迄含小數點；抓取範圍由 180 秒改為 181 秒
2 重複執行模型出現無解 模型求解後未清空資料；單次求解後將所有資料清空回到初始狀態
3 RSI Record 檔名重複 前處理未考量未排程活動；檔名處理統一移至排程後再進行
4 模型排程結果不符預期 RSI PBK 過於分散；調整目標式權重、限制式寫法及 gap 值
5 GAT / GOHOME 未考量 Attitude 前處理確認兩活動間是否存在 Attitude Maneuver，並新增 RSI PBK 限制
6 執行效率與品質無法平衡 CPLEX 參數：8 Threads、Aggressive Cuts、Best-Estimate Node Search
7 跨 session 排程造成無解 GAT / GOHOME 只依 X-band 時間篩選；BigM 值設定 ≥ 1,000,000

PHASE 04 — REVISION 2024 / 09 / 25
1 addResourceChange 檔名檢查機制 排序未使用更新後 RSI Delete 時間；以更新後時間排序再建立活動
